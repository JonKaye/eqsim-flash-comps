stateMgr

Description

The state (network) manager class is used at run-time to manage which state(s) are active and which have been visited last (the history mechanism), in each network (collection of states).

You must create a state manager for each hierarchical state (hState), and you may create several for concurrent hierarchical states (hStateC), one to manage each sub-network.

Usage

new stateMgr(name, id, defaultState, containerState);

Parameters

name String name of the manager. Used in trace output

id Typically alphabetic, this is the ID of the manager (must be unique among other managers in the same hStateC)

defaultState ID of the default start state for this network

containerState Pointer to the state that contains this network.

Example

// For hState's
s2 = new hState("Time State", 0, theTopStateMgr);
timeSM = new stateMgr("Time Mgr", "A", 1, s2); // default start state is ID 1

// For hStateC's
s2 = new hStateC("Big State", 0, theTopStateMgr);
// default start state for first sub-network is ID 1
timeSM = new stateMgr("Time Mgr", "A", 1, s2);
// default start state for second sub-network is ID 5 
dateSM = new stateMgr("Date Mgr", "B", 5, s2);

Code Hint

Variable names with the suffix "_stMgr" are recognized for the sake of codehinting, as state managers..
