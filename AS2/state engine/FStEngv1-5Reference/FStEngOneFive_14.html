chgSt

Usage

myhState.chgSt(transID, val)

Parameters

transID Transition ID

val (optional) Value to accompany the transition, if needed.

Returns

Nothing

Description

Note: In version 1.5, the transition triggering is handled automatically, so the developer does not need to use this method if a trigger name is used when the transition is created.

Change state using transition transID, with an optional value for the transition function as a second argument. If you look at the code, this function passes the information to the chgSt method of the state manager to do the real work. The reason we define this dummy is so that modelers see the direct relationship to the transition id, which is unique by state. For example, if a caller wants to change the current state, state 2, using transition 0, the caller would write:

state2.chgSt(0);

This implies that the transition 0 is relative to the state. The same unaltered call would be

sm.chgSt(0);

in which 'sm' is the state manager of state 2 and its siblings. It is less obvious from the second form that the processor is using the current state (2)'s transition 0, as opposed to, perhaps, state 1 or 6's transition with id 0.

The second argument, val, is an optional value to send in to the transition function.
