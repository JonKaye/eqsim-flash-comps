transLvl

Description

The transLvl is a transition that links two states (source and target) not in the same layer of the network, i.e., not sibling states. To traverse this transition requires knowing the least common ancestor (LCA) of the two states, and then the path from the LCA down to the target (or history).

Usage

new transLvl(fn, upLevels, statePath, histFlag);

Parameters

fn Function to execute when this transition fires

upLevels Number of levels to ascend to the LCA. If this transition goes into a sub-state of the current state, this value is set to -1.

statePath An array of state ID's on the path from the LCA to the target

histFlag Boolean value indicating whether to use the last state visited (history) after entering the last state in the state path, or not (default is false)

Example

// LCA is 2 levels up, target is 2 levels up and 1 state down from there (ID #3)
myState.addTrans(0, new transLvl(myTransFn, 2, [ 3 ]), "onButtonPress");

Code Hint

Variable names with the suffix "_transLvl" are recognized for the sake of codehinting, as multi-level transitions.
