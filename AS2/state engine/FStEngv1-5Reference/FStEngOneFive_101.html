trans

Description

The simple transition is one from a source state to a target sibling state (i.e., at the same level in the hierarchical network). Simple transitions have a completely specified source and target, though the target can be the history node (last state visited, determined at transition time). Simple transitions are distinguished from multi-level transitions, or transLvl.

Usage

new trans(fn, targetStateID, histFlag);

Parameters

fn Function to execute when this transition fires

targetStateID ID of the target state, if not using the history mechanism

histFlag Boolean value indicating whether to use the last state visited (history). If this is true (it is not, by default), then the targetStateID is ignored.

Example

// This is a transition from myState to its sibling (identified as #2)
myState.addTrans(0, new trans(myTransFn, 2));

Code Hint

Variable names with the suffix "_trans" are recognized for the sake of codehinting, as simple transitions.
